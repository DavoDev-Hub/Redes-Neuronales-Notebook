1.4 Anlisis de la variable objetivo (Salary Estimate)

La variable Salary Estimate contiene rangos de sueldo expresados en miles de dlares (K) y, en algunos casos, se indica si el sueldo es por hora o provisto por el empleador. Para analizar numricamente esta variable, definimos una funcin que extrae el salario mnimo y mximo de cada rango, eliminando textos adicionales, y calcula un salario promedio.

In[93]:
# Definir funcin de parseo de salarios ()
def parse_salary(s):
    s = str(s)
    if s == '-1' or s.lower() == 'nan':
        return np.nan, np.nan, np.nan
    # Eliminar textos auxiliares
    s = s.replace('Employer Provided Salary:', '')
    s = s.replace('(Glassdoor est.)', '')
    s = s.replace('(Employer est.)', '')
    s = s.replace('$', '').replace(',', '')
    s_lower = s.lower()
    hourly = 'per hour' in s_lower
    s_lower = s_lower.replace('per hour', '')
    # Separar mnimo y mximo
    if '-' in s_lower:
        low_str, high_str = s_lower.split('-')
        try:
            low = float(low_str.strip().replace('k', '')) * 1000
            high = float(high_str.strip().replace('k', '')) * 1000
        except ValueError:
            return np.nan, np.nan, np.nan
        # Convertir a anual si es por hora (aprox. 40 horas semana * 52 semanas)
        if hourly:
            low = low / 1000
            high = high / 1000
            low *= 40 * 52
            high *= 40 * 52
        avg = (low + high) / 2
        return low, high, avg
    return np.nan, np.nan, np.nan

Tras cargar el dataset, la columna Salary Estimate contiene rangos salariales expresados en miles de dlares (por ejemplo, $53K$91K (Glassdoor est.)) e incluso algunos sueldos por hora o marcados como provistos por el empleador. Para poder trabajar con esos valores de forma cuantitativa, se define la funcin parse_salary que realiza lo siguiente:

  1. Deteccin de valores faltantes: si la cadena es "-1" o "nan", devuelve (NaN,NaN,NaN) para indicar que no hay salario vlido.
  2. Limpieza de texto: elimina trminos auxiliares como EmployerProvidedSalary:, (Glassdoor est.) o (Employer est.) que aparecen en las estimaciones de Glassdoor.
  3. Identificacin de sueldos por hora: convierte la cadena a minsculas y comprueba si contiene la frase "perhour". En ese caso marca la variable hourly = True y elimina esa frase del texto para facilitar el parseo.
  4. Extraccin del rango: divide la cadena por el guion "-" para separar el mnimo y el mximo. A cada parte le elimina la letra k y la convierte a nmero, multiplicando por1000 para pasar de miles a unidades.
  5. Conversin a salario anual: si el sueldo era por hora (hourly = True), multiplica los valores por 40 52 (40horas semanales52semanas) para estimar el ingreso anual.
  6. Clculo del promedio: calcula la media entre el salario mnimo y el mximo.
  7. Retorno: devuelve una tupla (min_salary, max_salary, avg_salary). Si no hay rango vlido, devuelve (NaN,NaN,NaN).

Posteriormente se aplica esta funcin a cada fila de la columna SalaryEstimate con data['Salary Estimate'].apply(...) y se crean tres nuevas columnas: min_salary, max_salary y avg_salary. La tabla resultante muestra estas columnas junto a la estimacin original para comprobar que el parseo es correcto. Por ejemplo, el rango $53K$91K (Glassdoor est.) se convierte en un salario mnimo de53000USD, un mximo de91000USD y un salario promedio de72000USD. Estas columnas numricas sern la base para el anlisis estadstico y el entrenamiento del modelo en las siguientes etapas del proyecto.
